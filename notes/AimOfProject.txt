I want to  built an Enterprise Order Management System using Spring Boot microservices. 
The system has to be separate services for authentication, users, products, inventory, orders, payments, notifications, and invoices. 
I want to  use API Gateway for routing and JWT validation, Eureka for service discovery, 
Config Server for centralized configuration, Feign for synchronous communication, Kafka for asynchronous events, and Resilience4j 
for circuit breaker and fallback handling. Each service owns its own database, and order
 processing follows the Saga pattern for distributed transaction management. 
I also want dockerize the services and prepare Kubernetes manifests for deployment and scaling.
